High-level theory — enough to orient your brain

Big-O

Big-O describes how work grows as input grows. Common exam order: O(1) < O(log n) < O(n) < O(n log n) < O(n²).

ADT vs implementation

Stack, Queue, Priority Queue describe behavior. Array, Linked List, Heap are ways to implement behavior.

Worst vs average vs amortized

Average is typical. Worst is guaranteed bad case. Amortized means a sequence of operations is cheap overall, even if one operation is expensive.

Stability + in-place

Stable preserves equal-item order. In-place uses little extra memory. These are common sorting comparison questions.
